Cách sử dụng TypeORM API để tương tác với cơ sở dữ liệu MySQL 
Đầu tiên, hãy nhập các Entities (Bảng) và AppDataSource từ file src

1. INSERT INTO 
Chỉ cần sử dụng new để tạo một đối tượng mới 
Ví dụ: Const Users = new User() - Table 
Sau đó chỉ cần sử dụng thuộc tính để xác định dữ liệu 
Ví dụ: Users.Name = 'Tuan' - Name là thuộc tính của User 
Cuối cùng, lưu nó bằng Entity manager với đoạn mã sau 
Ví dụ: await AppDataSource.manager.save(Users)

2. SELECT FROM 
Trước khi sử dụng SELECT, đầu tiên tạo Repositories với Entities 
Ví dụ: const UserRepositories = AppDataSource.getRepository(User) 
Sau đó, tùy thuộc vào dữ liệu tìm kiếm, chúng ta có thể sử dụng mã:
    Tìm đối tượng đầu tiên khớp với danh mục: const firstUser = await UserRepositories.findOneby({id: 1})
    Tìm tất cả đối tượng trong Entities: const Users = await UserRepositories.find()
    Tìm dữ liệu khớp với danh mục: const CodingUser = await UserRepositories.findBy({work: Coding})
    Tìm tất cả đối tượng trong Entities và đếm số lượng: const [Users, UsersCount] = await UserRepositories.findAndCount()

3. DELETE FROM 
Đầu tiên, tìm kiếm dữ liệu cần xóa 
Ví dụ: const firstUser = await UserRepositories.findOneby({id: 1}) 
Sau đó sử dụng mã remove để xóa nó 
Ví dụ: await UserRepositories.remove(firstUser)

4. UPDATE 
Đầu tiên, tìm kiếm dữ liệu cần cập nhật 
Ví dụ: const firstUser = await UserRepositories.findOneby({id: 1}) 
Sau đó thay đổi thuộc tính cần cập nhật 
Ví dụ: firstUser.work = Sleeping 
Lưu lại = Cập nhật dữ liệu Ví dụ: await AppDataSource.manager.save(Users)

Cập nhật cách tạo bảng - file JavaScript not TypeScript
Lưu ý: Khi import file, lúc import các entities, hãy bỏ dấu ngoặc vì đó là module chứ ko phải thực thể riêng
Cấu trúc của các Entities như mẫu, tuy nhiên ta có các đặc điểm sau: 
    name: Tên module đại diện cho table (Entities)
    tableName: Tên table ở MySQL
    columns: các cột thông tin - type như trong MySQL, tuy nhiên nếu các table có khóa thì sẽ có thuộc tính primary
cũng như nếu muốn tự tạo theo kiểu tăng tiến thì dùng generated: "increment"
    relations: Mối liên kết, cái này nếu muốn thể hiện mỗi liên kết giữa các bảng nếu tạo trong đây mà ko tạo trước  
trong MySQL. Các thuộc tính khác như sau: 
        target: tên table kết nối
        type: Loại liên kết
        joinColumn: true, // Đánh dấu User là bảng chủ chứa khóa ngoại
        cascade: cascade trong MySQL (đừng quan tâm)
